# 모임 이벤트 관리 웹 MVP 개발 로드맵

카카오톡에 묻히는 공지/참여 확인을 한 곳에서 관리하여 소규모 모임 주최자의 운영 부담을 줄이는 웹 서비스

## 개요

모임 이벤트 관리 웹은 수영, 헬스 등 소규모 동호회나 지인 모임을 이끄는 **주최자**와, 공유 링크로 간단히 참여 응답하는 **참여자**를 위한 서비스로 다음 기능을 제공합니다:

- **이벤트 관리**: 이벤트 생성/조회/수정/삭제 및 공유 링크 생성
- **참여 응답**: 비회원도 링크만으로 참여 응답 가능, 대기 명단 자동 관리
- **공지사항**: 이벤트별 공지 등록/수정/삭제 및 고정 기능
- **실시간 명단**: 참여자 현황 실시간 조회 및 주최자 관리 기능

### 기술 스택

- **프론트엔드**: Next.js 15 (App Router), React 19, TypeScript
- **UI**: shadcn/ui + Tailwind CSS
- **백엔드/인증**: Supabase (Auth, Database, Realtime)
- **배포**: Vercel

## 개발 워크플로우

1. **작업 계획**
   - 기존 코드베이스를 학습하고 현재 상태를 파악
   - 새로운 작업을 포함하도록 `ROADMAP.md` 업데이트
   - 우선순위 작업은 마지막 완료된 작업 다음에 삽입
   - UI/UX를 더미 데이터로 먼저 완성한 뒤, 검토 결과를 반영하여 DB 스키마를 확정한다

2. **작업 생성**
   - 기존 코드베이스를 학습하고 현재 상태를 파악
   - 고수준 명세서, 관련 파일, 수락 기준, 구현 단계 포함
   - API/비즈니스 로직 작업 시 "테스트 체크리스트" 섹션 필수 포함 (Playwright MCP 테스트 시나리오 작성)

3. **작업 구현**
   - 작업 파일의 명세서를 따름
   - 기능과 기능성 구현
   - API 연동 및 비즈니스 로직 구현 시 Playwright MCP로 테스트 수행 필수
   - 각 단계 후 작업 파일 내 단계 진행 상황 업데이트
   - 구현 완료 후 Playwright MCP를 사용한 E2E 테스트 실행
   - 각 단계 완료 후 중단하고 추가 지시를 기다림

4. **로드맵 업데이트**
   - 로드맵에서 완료된 작업을 완료 표시로 갱신

## 개발 단계

### Phase 1: 애플리케이션 골격 구축

- [ ] **Task 001: 프로젝트 구조 및 라우팅 설정** - 우선순위
  - Next.js App Router 기반 이벤트 관련 전체 라우트 구조 생성
    - `/events` (이벤트 목록, 로그인 필요)
    - `/events/new` (이벤트 생성 폼)
    - `/events/[id]` (이벤트 상세 공개 페이지)
    - `/events/[id]/manage` (주최자 관리 페이지)
  - 각 라우트에 빈 페이지 컴포넌트 파일 생성
  - 공통 레이아웃 컴포넌트 골격 구현 (헤더, 네비게이션, 푸터)
  - 홈페이지(`/`) 서비스 소개 랜딩 페이지 골격 구성

- [ ] **Task 002: 타입 정의 및 더미 데이터 설계**
  - TypeScript 인터페이스 정의 (`types/` 디렉토리)
    - `Event` 타입: id, organizer_id, title, description, location, event_date, rsvp_deadline, max_attendees, share_token, created_at
    - `Announcement` 타입: id, event_id, title, content, is_pinned, created_at
    - `Attendee` 타입: id, event_id, user_id(nullable), guest_name(nullable), status(attending/not_attending/maybe/waiting), memo, created_at
  - API 응답/요청 타입 정의
  - 더미 데이터 생성 유틸리티 작성 (`lib/dummy-data.ts`)

  - 더미 데이터 파일 작성 (이벤트, 공지사항, 참여자 목록)

- [ ] **Task 004: 모든 페이지 UI 완성 (더미 데이터 기반)**
  - 홈페이지(`/`) UI 구현
    - 서비스 소개 히어로 섹션
    - 주요 기능 설명 카드
    - "시작하기" CTA 버튼
  - 이벤트 목록 페이지(`/events`) UI 구현

    - 빈 상태 안내
  - 이벤트 생성 페이지(`/events/new`) UI 구현

    - 폼 유효성 검사 UI

    - 이벤트 정보 표시 영역
    - 공지사항 섹션 (고정 공지 상단 배치)
    - 참여 응답 폼 (이름 입력 + 상태 선택)

  - 라이트/다크 모드 대응

### Phase 3: 핵심 기능 구현

- [ ] **Task 005: 데이터베이스 스키마 확정 및 마이그레이션 적용** - 우선순위
  - Phase 2 UI/UX 검토 결과를 반영하여 최종 데이터베이스 스키마 확정

    - `attendees`: 비회원도 생성 가능, 주최자만 수정/삭제 가능

- [ ] **Task 006: 인증 시스템 정비 (Google OAuth)**

  - 인증 상태에 따른 라우트 보호 미들웨어 구현

  - 로그아웃 기능 연동

- [ ] **Task 007: 이벤트 CRUD API 및 연동**
  - Server Actions 또는 API Route 구현

    - `GET /events/[id]` - 이벤트 상세 조회 (share_token 기반 공개 접근)
